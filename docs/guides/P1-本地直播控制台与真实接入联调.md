# P1 — 本地直播控制台与真实接入联调

> 更新时间: 2026-04-19
> 适用范围: 本地数字人演示页升级、AIGCPanel / 飞书 / 弹幕源联调

## 1. 你会得到什么

运行 `python scripts/live_bridge.py` 后，会启动一个本地直播控制台，统一展示：

1. 数字人舞台区
2. 当前直播脚本
3. 实时弹幕流与投诉告警
4. AIGCPanel、飞书、弹幕源状态
5. 本场复盘生成入口

默认地址：

```text
http://127.0.0.1:8765
```

## 2. 启动步骤

1. 先准备配置文件。

```bash
copy config\live-bridge.example.json config\live-bridge.json
```

2. 如果只想先看本地完整演示，直接运行：

```bash
python scripts/live_bridge.py --load-demo
```

3. 浏览器打开：

```text
http://127.0.0.1:8765
```

## 3. 飞书接入

你可以用两种方式配置飞书 Webhook：

1. 在 `config/live-bridge.json` 的 `integrations.feishu.webhook_url` 直接填写
2. 或设置环境变量 `FEISHU_WEBHOOK_URL`

控制台里的“发送飞书测试”按钮会走真实 Webhook。

投诉弹幕进入 D 类时，也会自动触发同一条真实告警链路。

## 4. AIGCPanel 接入

本项目现在接的是 **AIGCPanel launcher API** 方式，不再只是静态素材演示。

按 AIGCPanel 官方 launcher 文档，当前联调用到四个接口：

1. `/ping`
2. `/submit`
3. `/query`
4. `/cancel`

配置位置在 `config/live-bridge.json`：

```json
"aigcpanel": {
  "enabled": true,
  "base_url": "http://127.0.0.1:8888",
  "probeBaseUrls": [
    "http://127.0.0.1:8888",
    "http://127.0.0.1:3030"
  ]
}
```

字段说明：

1. `base_url` 会优先按本机 launcher 默认端口 `8888` 探测
2. `probeBaseUrls` 会在 `8888` 不通时继续尝试 `3030`
3. 如果远端 `/config` 已提供 launcher 配置，可以不再手工写死 `entry` / `entryArgs`
4. `${TEXT}`、`${HOST_NAME}`、`${PRODUCT_NAME}`、`${ROOM_TITLE}`、`${SESSION_ID}` 仍会在提交时自动注入
5. `root`、`envs`、`resultRoots` 可选，用来指定工作目录、环境变量和允许回放的本地结果目录
6. 开启 `auto_query` 后，bridge 会在提交任务后自动轮询，拿到真实音频/视频结果就直接回显到直播页

如果你的 AIGCPanel launcher 端口既不是 `8888` 也不是 `3030`，直接改 `base_url`。

如果 AIGCPanel 和 BarrageGrab 都默认占用 `8888`，一定要手动错开端口，避免冲突。

### 4.1 首次装机的关键提醒

如果你是刚重装 AIGCPanel，或者更换过电脑，一定要先在 AIGCPanel 里确认以下资产已经存在：

1. 声音克隆记录
2. TTS 可用模型
3. 数字人视频模板

如果这些记录不存在，本地 bridge 虽然已经接入成功，但不会得到“原来的真人音色效果”，因为 AIGCPanel 根本没有可调用的原始音色和模板。

## 5. BarrageGrab / 弹幕源接入

控制台桥的收口接口是：

```text
POST /api/barrage
```

如果你使用 BarrageGrab 风格的 WebSocket 弹幕输出，可以直接运行：

```bash
python scripts/connect_douyin_barragegrab.py --bridge-url http://127.0.0.1:8765/api/barrage --websocket-url ws://127.0.0.1:8888
```

说明：

1. 这个脚本默认按 `barragegrab_type3` 解析评论事件
2. 依赖 `websocket-client`
3. 收到弹幕后会转成 `{user, message}` 再打进本地桥
4. 控制台里的“弹幕源”状态会随着 relay 心跳更新

## 6. 常用操作流

### 6.1 纯本地演示

1. 启动 `python scripts/live_bridge.py --load-demo`
2. 打开控制台
3. 查看数字人舞台、弹幕流、告警和复盘

### 6.2 接真实飞书

1. 配置 `FEISHU_WEBHOOK_URL`
2. 控制台点击“发送飞书测试”
3. 再手动输入一条投诉弹幕，例如“质量太差了”

### 6.3 接真实 AIGCPanel

1. 在配置里启用 `integrations.aigcpanel.enabled`
2. 填好 `base_url`、`entry`、`entryArgs`
3. 控制台依次点击 `Ping`、`提交当前播报`、`查询任务`

### 6.4 接真实弹幕源

1. 先启动本地 bridge
2. 再启动 BarrageGrab
3. 再运行 `scripts/connect_douyin_barragegrab.py`
4. 看控制台右侧弹幕流是否滚动

## 7. 当前边界

这版已经从“静态演示页”升级成“本地直播控制台 + 真实接入桥”，但仍有两个边界要注意：

1. AIGCPanel 具体执行的 `entry` / `entryArgs` 仍然要按你本机实际工作流填写
2. BarrageGrab 的 WebSocket 协议如果不是 `Type=3 / Data.User.NickName / Data.Content` 这一套，需要在 relay 脚本里调整归一化逻辑
