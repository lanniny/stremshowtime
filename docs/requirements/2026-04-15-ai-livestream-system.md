# AI数字直播带货体系 — 需求文档

> 冻结日期: 2026-04-15
> 状态: FROZEN

## 1. 项目概述

为抖音电商直播场景搭建 AI 数字直播体系，实现从直播准备、脚本生成、数字人驱动、弹幕互动到数据复盘的全流程自动化。

## 2. 核心约束

| 约束项 | 决策 |
|--------|------|
| 团队技术能力 | 无开发者，全靠 AI 辅助 + 低代码/SaaS 搭建 |
| 数字人形象 | 必须基于团队真人成员克隆 |
| 弹幕抓取 | 通过第三方工具实现 |
| TTS语音 | 无偏好，跟随数字人平台内置方案 |
| 部署平台 | **OpenClaw**（开源AI Agent，本地部署） |
| 上线时间 | 尽快 MVP |
| 主打商品 | "一川桑语" 桑葚汁系列（果汁 + 果酒） |

## 3. 功能需求

### 3.1 直播前 — 脚本生成

- **输入**: 商品名称/关键词
- **处理**: 从商品信息库读取数据，LLM 生成直播脚本
- **输出**: 包含5个环节的完整口播脚本
  1. 开场预热（欢迎语、福利预告）
  2. 痛点挖掘（2-3个场景）
  3. 产品展示与卖点讲解
  4. 互动答疑与逼单
  5. 下播预告

### 3.2 直播中 — 实时互动

- **弹幕抓取**: 第三方工具实时获取弹幕流
- **弹幕分类**: 5类（A产品咨询/B购买意向/C闲聊/D投诉/E无关）
- **自动回复**: A-C类自动回复，D类安抚+转人工，E类忽略
- **语音播报**: 回复文本 → TTS → 数字人口播
- **响应时间**: 0.5-1秒内判断弹幕类型

### 3.3 直播后 — 数据复盘

- 自动统计: 时长、观看人次、在线峰值、粉丝增长、互动数据
- 弹幕词云: 高频词分析
- 优化建议: 基于数据给出1-2条改进建议

### 3.4 行为边界（红线）

- 禁止政治敏感、低俗内容
- 禁止恶意贬低竞品
- 禁止泄露隐私
- 禁止未经证实的功效承诺
- 超出范围的问题必须转人工

## 4. 非功能需求

- 低代码/零代码实现优先
- 月度运营成本可控
- 可扩展至更多商品品类
- 操作界面对非技术人员友好

## 5. 验收标准

- [ ] 输入商品名 → 30秒内生成完整直播脚本
- [ ] 数字人形象与团队成员相似度达到可用水平
- [ ] 弹幕回复延迟 < 3秒（含TTS生成）
- [ ] D类投诉弹幕能正确触发人工通知
- [ ] 直播结束后自动生成复盘报告
