## 一、为什么选 OpenClaw？

| 对比维度 | OpenClaw | Coze（扣子） |
|---------|----------|-------------|
| 费用 | **开源免费** | 免费额度有限，超出收费 |
| 自由度 | 完全可定制 Skill | 受限于平台能力 |
| LLM 选择 | 可接任意模型（GPT/Claude/通义/DeepSeek） | 绑定字节系模型 |
| 本地运行 | **支持**，数据不出服务器 | 云端，数据经第三方 |
| 自动化深度 | RPA + API + Shell 全链路 | 仅限 Bot 对话 + 插件 |
| 弹幕集成 | 可直接 RPA/OCR 抓取，或对接第三方API | 需外部 Webhook 桥接 |
| 消息通知 | 原生支持飞书/钉钉/微信 | 需插件配置 |
| 社区生态 | 3000+ Skills，200+/月增长 | 插件市场相对封闭 |

**结论**: OpenClaw 更灵活、更省钱、更适合全流程自动化场景喵～

### 数据流详解

```
【直播前】
运营人员 → 飞书/对话 发送"准备XX商品直播"
  → OpenClaw 触发 Skill 1
  → 读取商品知识库
  → 调用 LLM 生成脚本
  → 输出完整口播文案 → 推送给数字人平台

【直播中】
抖音弹幕 → 第三方抓取工具 → OpenClaw 触发 Skill 2
  → LLM 弹幕分类 (A/B/C/D/E)
  → A-C: 生成回复 → 公屏文字回复 + TTS播报
  → D: 安抚回复 + 飞书通知人工
  → E: 静默忽略

【直播后】
运营人员 → 发送"复盘本场直播"
  → OpenClaw 触发 Skill 3
  → 汇总弹幕日志 + 抖店数据
  → LLM 生成复盘报告 + 优化建议
```

**SKILL.md 内容:**

```markdown
---
name: livestream-script
description: 根据商品信息生成抖音直播带货口播脚本，包含5个标准环节
user-invocable: true
---

# 直播脚本生成器

## 触发方式
用户发送: "准备 [商品名称] 的直播" 或 "/直播脚本 [商品名称]"

## 执行步骤

1. 从 `{baseDir}/references/products.json` 读取商品信息库
2. 匹配用户指定的商品，提取全部字段:
   - 商品名称、核心卖点(至少3条)、价格与优惠、规格参数、
     配料表、库存、用户痛点、产品故事、购买链接、竞品对比
3. 调用 LLM 生成直播脚本，严格包含以下5个环节:

### 环节① 开场预热（约1分钟话术）
- 欢迎语、主播自我介绍
- 本场福利预告（价格优惠、赠品等）
- 引导点关注、亮灯牌

### 环节② 痛点挖掘（约1-2分钟）
- 模拟2-3个观众日常痛点场景
- 引出本商品作为解决方案
- 使用"你是不是也遇到过..."的共情话术

### 环节③ 产品展示与卖点讲解（约3-5分钟）
- 核心卖点转化为口语化、有画面感的描述
- 穿插竞品对比（不贬低，只对比数据）
- 突出性价比

### 环节④ 互动答疑与逼单（约2-3分钟）
- 主动抛出互动问题（"想要的扣1"）
- 强调限时优惠、库存告急
- 倒计时催单话术

### 环节⑤ 下播预告（约30秒）
- 感谢陪伴
- 预告下一场直播
- 引导关注不迷路

## 风格要求
- 称呼: "家人们""宝宝们""兄弟姐妹们"
- 语气: 热情活泼，多感叹号，语速适中偏快
- 禁止: 政治敏感、低俗内容、贬低竞品、虚假承诺

## 输出格式
输出一段可直接口播或推送给数字人的完整脚本文案，每个环节用标题分隔。
```



**SKILL.md 内容:**

```markdown
---
name: barrage-responder
description: 抖音直播间弹幕实时监控、智能分类与自动回复
user-invocable: true
---

# 弹幕监控与回复助手

## 触发方式
- 用户发送: "/弹幕助手 启动" 进入监控模式
- 接收到弹幕文本时自动处理

## 弹幕分类规则（0.5-1秒内判断）

### A类: 产品咨询
**特征**: 询问价格、成分、规格、保质期、口感等
**典型弹幕**: "多少钱？" "配料表有什么？" "保质期多久？" "好喝吗？"
**处理**:
1. 从商品知识库精准提取答案
2. 语气热情亲切
3. 末尾附加: "点击下方小黄车1号链接就可以拍啦～"
**回复上限**: 50字以内

### B类: 购买意向/催单
**特征**: 表达已购买、求优惠、询问库存
**典型弹幕**: "已拍，加急发货" "还有库存吗？" "给个优惠"
**处理**:
1. 肯定购买行为: "感谢家人支持！"
2. 营造紧迫感: "库存不多了，拼手速哦～"
3. 引导下单

### C类: 闲聊/感谢
**特征**: 打招呼、夸赞、非产品相关闲聊
**典型弹幕**: "讲得真好" "从XX视频过来的" "主播真漂亮"
**处理**:
1. 礼貌互动，维持直播间热度
2. 引导关注: "觉得好的点点关注不迷路哦～"

### D类: 负面/投诉
**特征**: 抱怨质量、物流、客服等
**典型弹幕**: "质量太差了" "物流太慢" "客服不理人"
**处理**:
1. 立即发送安抚话术: "非常抱歉给您带来不好的体验，联系在线客服小姐姐优先处理哦～"
2. 通过飞书 Webhook 发送预警消息给人工场控:
   ```
   ⚠️ 直播间投诉预警
   用户: [昵称]
   内容: [弹幕原文]
   时间: [当前时间]
   建议: 尽快联系用户处理
   ```

### E类: 无关/刷屏
**特征**: 竞品广告、恶意刷屏、完全无关话题
**处理**: 不做任何回复，静默忽略

## 行为红线
- 禁止讨论政治敏感、色情低俗、封建迷信
- 禁止恶意贬低竞品
- 禁止泄露主播个人隐私
- 禁止做未经证实的功效承诺
- 超出知识库范围的高风险问题 → 回复"具体可以咨询客服小姐姐哦～"
```

**串联方式:**

```

路径B（保底 — 半自动）:
```

## 九、风险与应对

| 风险 | 影响 | 应对方案 |
|------|------|---------|
| 数字人效果不自然 | 观众不信任 | 先"真人出镜+AI弹幕回复"模式过渡 |
| 弹幕回复延迟>3秒 | 互动体验差 | 优先文字回复，TTS播报作为增强 |
| 弹幕工具被平台封禁 | 链路中断 | 准备2-3个备选工具 |
| LLM 回复不当 | 违规风险 | Skill 中设置严格红线+关键词过滤 |
| OpenClaw 部署困难 | 进度受阻 | 保底方案: 降级为 Coze 实现 |
| 抖音限制数字人直播 | 无法开播 | 关注政策，保持真人+AI混合能力 |
